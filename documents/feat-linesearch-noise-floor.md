# Предельная достижимая точность расчёта в методе Ньютона

При итерационном решении нелинейных уравнений (например, задач гидравлики нефтегазовой отрасли) необходимо различать два уровня точности: погрешность предметной области и погрешность численного расчёта.

**Погрешность предметной области.** С точки зрения прикладной задачи всегда можно оценить порядок величин, которые считаются малыми. Для объектов нефтегазовой отрасли характерные давления от объекта к объекту варьируются в меньшем диапазоне, чем характерные расходы. Критерий сходимости, сформулированный в терминах предметной области, оперирует конечной целевой точностью с точки зрения пользователя. Эта погрешность должна быть заведомо выше погрешности численного расчёта в формате `double`.

**Граница численной точности.** При очень высоких требованиях к точности целевые функции могут потерять гладкость и стать нефизичными. Требование на малое приращение аргумента (давления) может приводить к тому, что соответствующее приращение значения целевой функции (расхода) окажется ниже порога разрешимости вещественной арифметики `double`. 

Гипотеза - машинная точность `double` (порядка 1e-16) деградирует с ростом числа арифметических операций; снижение числа операций при вычислении целевой функции через невязку потенциально позволяет приблизить фактическую точность к теоретическому пределу.

**Дополнительный критерий на величину целевой функции.** Разумно ввести порог `function_target_value` — параметр задачи, который по своей сути является соотношением сигнал/шум (типовое значение 1e-8). Нарушение порога свидетельствует о выходе за предел точности по целевой функции.

Проверка выполняется в самом начале `search`, до первого вычисления `f(alpha)` и `f(beta)`, и применяет следующие правила:

- если оба значения `f_a` и `f_b` превышают порог — обработка не требуется, итерации ЗС запускаются штатно;
- если одно из значений ниже порога — возвращается меньшее из двух как текущий минимум;
- если оба значения ниже порога — предельная точность уже достигнута, `search` немедленно возвращает `alpha = 0` (шаг Ньютона не делается).

**Реализация.** Логика оформлена методом `golden_section_parameters::try_resolve_step_by_target_value(f_a, f_b, a, b)` (не модификация `check_convergence`, который работает с одним значением и заточен под внутренний цикл). Метод возвращает `std::optional<double>`:

- `std::nullopt` — порог не достигнут, поиск продолжается штатно;
- значение — alpha, который `search` возвращает немедленно без запуска итераций.

Такой интерфейс самодокументируем: наличие значения означает «ответ уже известен». Накладные расходы `std::optional<double>` пренебрежимо малы на фоне стоимости вычисления целевой функции.

Критерий предотвращает дальнейшее итерирование в области, где численный шум доминирует над полезным сигналом, и позволяет корректно завершить расчёт при достижении предельной достижимой точности.
